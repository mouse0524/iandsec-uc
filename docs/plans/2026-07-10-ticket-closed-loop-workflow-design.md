# 工单闭环流转设计

## 目标

基于当前工单逻辑，把现网问题和现网需求都留在系统内闭环处理，去掉 Redmine 同步依赖。

新增角色：

- 产品
- 测试
- 研发

保留现有角色：

- 用户
- 渠道商
- 技术
- 客服
- 管理员

## 当前逻辑

当前 `Ticket` 是客户/现场侧工单主表，已有状态主线：

提交 -> 客服审核 -> 技术处理 -> 现场验证/待关闭 -> 完成

当前缺口：

- 产品、测试、研发不是内置角色。
- 产品评估、测试过滤、研发处理、测试验证没有独立状态。
- Redmine 同步仍存在于模型字段、API、设置页、技术处理页和后台定时任务中。

## 设计原则

- `Ticket` 继续作为外部工单主表，不把产研细节都塞进去。
- 新增一个轻量内部处理任务层，替代 Redmine 承接产品、测试、研发流转。
- 第一阶段只让流程闭环，不做可配置工作流引擎。
- Redmine 先从业务流、界面、接口和定时任务中移除；数据库旧字段可后续迁移清理。

## 状态设计

`TicketStatus` 扩展为：

- `pending_review`：待客服审核
- `cs_rejected`：客服驳回
- `tech_processing`：技术处理
- `test_filtering`：测试过滤
- `product_evaluation`：产品评估
- `rd_processing`：研发处理/开发
- `test_verification`：测试验证
- `field_verification`：现场验证
- `done`：完成

兼容处理：

- `pending_close` 不再作为新流程必经状态，新流程由现场验证通过后直接完成。
- `tech_rejected` 可保留兼容历史数据，新流程优先通过测试/产品/现场退回路径表达。

## 内部任务设计

新增内部任务表，字段保持最小：

- `ticket_id`：关联工单
- `stage`：`product_evaluation` / `test_filtering` / `rd_processing` / `test_verification`
- `assignee_id`：处理人
- `status`：`pending` / `processing` / `passed` / `rejected`
- `comment`：处理备注
- `created_by`：创建人
- `finished_at`：完成时间

`Ticket` 保存当前大状态和已有外部侧字段；内部任务保存产品、研发、测试经办人与处理记录。

## 现网问题流程

主线：

用户/渠道商/技术提交 -> 客服审核 -> 技术处理 -> 测试过滤 -> 研发处理 -> 测试验证 -> 用户/渠道商/技术现场验证 -> 完成

退回：

- 客服驳回：回提交人补充后重新提交。
- 测试过滤不通过：回技术处理。
- 测试验证不通过：回研发处理。
- 现场验证不通过：默认回测试验证。

## 现网需求流程

主线：

用户/渠道商/技术提交 -> 客服审核 -> 产品评估 -> 研发开发 -> 测试验证 -> 用户/渠道商/技术现场验证 -> 完成

退回：

- 客服驳回：回提交人补充后重新提交。
- 产品驳回：回提交人，说明不采纳原因。
- 研发无法实现或需澄清：回产品评估。
- 测试验证不通过：回研发开发。
- 现场验证不通过：默认回测试验证。

## 权限设计

- 用户/渠道商：提交、查看自己工单、补充、现场验证。
- 技术：提交、处理现网问题技术环节、查看自己负责或相关客户工单。
- 客服：审核、指派技术、查看工单。
- 产品：处理产品评估任务。
- 研发：处理研发任务。
- 测试：处理测试过滤和测试验证任务。
- 管理员：全量权限。

## 页面设计

保留现有页面：

- 工单提交
- 我的工单
- 客服审核
- 技术处理
- 工单详情

新增或改造：

- 产研任务页：产品、研发、测试共用列表，通过角色和任务阶段过滤可操作项。
- 工单详情展示内部任务流转记录。

不新增独立产品页、研发页、测试页，除非后续角色差异大到共用页变复杂。

## 去掉 Redmine

第一阶段移除：

- 后端 Redmine 定时任务启动。
- `/api/v1/ticket/redmine/push`
- `/api/v1/ticket/redmine/pull`
- `/api/v1/settings/redmine/metadata`
- 技术处理页 Redmine 同步按钮、弹窗、状态列。
- 设置页 Redmine 同步配置。
- 前端 API 中 Redmine 调用包装。

暂不强制物理删除：

- `Ticket` 上旧 Redmine 字段。
- 已存在的 Redmine 配置键。

这些旧字段和配置先不再展示、不再读写；后续需要干净 schema 时再做迁移删除。

## 最小验证

后端：

- 一条现网问题从提交走到完成。
- 一条现网需求从提交走到完成。
- 测试过滤不通过回技术。
- 产品驳回回提交人。
- 测试验证不通过回研发。
- 非对应角色不能处理对应内部任务。

前端：

- `pnpm.cmd run build`

基础检查：

- `python -m compileall app`
- 聚焦测试覆盖内部任务流转。
